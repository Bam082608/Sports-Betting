## üß≠ Protocol Decision Matrix

**Use this matrix to determine which protocols to apply for common scenarios:**

### Scenario: Multi-Game NHL Slate (5+ Games)
```
1. APEXVIPER_MULTI_GAME_TRIAGE ‚Üí Prioritize games
2. APEXVIPER_GAME_DAY_TIMELINE ‚Üí Check time allocation
3. APEXVIPER_GAME_ANALYSIS_PROTOCOL ‚Üí Deep-dive priority games
4. Specialized protocols as needed:
   - APEXVIPER_FATIGUE_MATRIX (if B2Bs present)
   - APEXVIPER_ALTITUDE_EDGE (if Colorado playing)
   - APEXVIPER_HOME_UNDERDOG (if home dogs present)
   - APEXVIPER_PUBLIC_MONEY (if heavy line movement)
5. APEXVIPER_GOALIE_CONFIRMATION ‚Üí Verify all starting goalies
6. APEXVIPER_LIVE_DATA_VERIFICATION ‚Üí Final injury/scratch check
7. APEXVIPER_PORTFOLIO_CONSTRUCTION ‚Üí Build portfolio
8. APEXVIPER_ADHD_EXECUTION_PROTOCOL_v3.2.0 ‚Üí Execute bets
```

### Scenario: Small Slate (1-3 Games)
```
1. APEXVIPER_GAME_DAY_TIMELINE ‚Üí Check time allocation
2. APEXVIPER_GAME_ANALYSIS_PROTOCOL ‚Üí Analyze all games
3. Specialized protocols as needed (Fatigue, Altitude, etc.)
4. APEXVIPER_GOALIE_CONFIRMATION ‚Üí Verify starting goalies
5. APEXVIPER_LIVE_DATA_VERIFICATION ‚Üí Final verification
6. APEXVIPER_PORTFOLIO_CONSTRUCTION ‚Üí Build portfolio
7. APEXVIPER_ADHD_EXECUTION_PROTOCOL_v3.2.0 ‚Üí Execute bets
```

### Scenario: Breaking News (Injury, Goalie Change, Scratch)
```
1. APEXVIPER_LIVE_DATA_VERIFICATION ‚Üí Verify news accuracy
2. APEXVIPER_INTEL_UPDATE_PROTOCOL ‚Üí Document and distribute
3. APEXVIPER_GOALIE_CONFIRMATION ‚Üí Re-verify if goalie related
4. Reassess affected bets using original analysis protocol
5. APEXVIPER_PORTFOLIO_RULES ‚Üí Check if portfolio adjustment needed
```

### Scenario: Losing Streak (3+ Consecutive Losses)
```
1. emergency-playbook.md ‚Üí Activate emergency procedures
2. emergency-protocols/stop-the-bleeding-protocol.md ‚Üí Full assessment
3. APEXVIPER_BANKROLL_MANAGEMENT ‚Üí Reassess unit sizes
4. Review tracking/COMPLETE-LESSONS-LEARNED.md ‚Üí Pattern check
5. APEXVIPER_ROUNDTABLE_PROTOCOL ‚Üí Multi-AI review session
6. Resume normal operations once cleared
```

### Scenario: Narrative/Storyline Bet Opportunity
```
1. APEXVIPER_NARRATIVE_BETTING_SYSTEM_v3.1.0 ‚Üí Identify narrative
2. APEXVIPER_GAME_ANALYSIS_PROTOCOL ‚Üí Validate with fundamentals
3. APEXVIPER_PUBLIC_MONEY ‚Üí Check if public on narrative side
4. Standard execution flow (Goalie Confirmation ‚Üí Portfolio ‚Üí Execution)
```
